# paolodona.com

Personal website of Paolo Donà. Plain static HTML/CSS with a small Node build step that turns markdown blog posts into HTML and regenerates the sitemap, RSS feed, and `llms.txt` index.

## Project structure

```
paolodona.com/
├─ index.html              # About (home)
├─ work.html               # Work history
├─ side-projects.html      # Side projects
├─ bookshelf.html          # Books I'm reading / have read
├─ blog.html               # Blog index — post list is regenerated by the build script
├─ 404.html
├─ robots.txt
├─ sitemap.xml             # ⚙ generated
├─ feed.xml                # ⚙ generated (RSS 2.0)
├─ llms.txt                # ⚙ generated (LLM-friendly site index)
├─ posts/                  # Source markdown for blog posts
│  └─ YYYY-MM-DD-slug.md
├─ blog/                   # ⚙ generated post HTML — one file per markdown post
├─ css/custom.css          # All styles
├─ img/
│  ├─ paolodona.png        # Headshot (used for hero + favicon + OG image)
│  └─ books/               # Book covers for the bookshelf grid
├─ scripts/build-blog.mjs  # Build script
└─ package.json
```

Files marked ⚙ are written by `npm run build` — don't edit them by hand.

## Local development

No build step is needed to view the site — it's plain HTML.

```sh
# Serve locally on http://localhost:8000
python -m http.server 8000
# or
npm run serve
```

Open `http://localhost:8000/` in a browser. Edit any HTML/CSS file and refresh.

## Adding a blog post

1. Create a new markdown file in `posts/` named `YYYY-MM-DD-slug.md` (the date prefix keeps files chronological in the folder; the actual publish date comes from the front-matter).
2. Add front-matter and body:

   ```markdown
   ---
   title: My post title
   date: 2026-05-12
   slug: my-post-title
   summary: One-sentence summary used on the blog index, RSS feed, and meta description.
   ---

   Markdown body goes here. Headings, links, code blocks, blockquotes and images all work.
   ```

3. Run the build:

   ```sh
   npm run build
   ```

4. Commit the new markdown file **and** the generated outputs (`blog/<slug>.html`, `blog.html`, `sitemap.xml`, `feed.xml`, `llms.txt`).

## What `npm run build` does

The script (`scripts/build-blog.mjs`) does five things in one pass:

1. **Renders each `posts/*.md`** through `marked` and `gray-matter` into a full HTML page in `blog/<slug>.html`. Each post page includes canonical, Open Graph, Twitter Card, and JSON-LD `BlogPosting` metadata.
2. **Updates `blog.html`** — replaces the contents between `<!-- posts:start -->` and `<!-- posts:end -->` with the latest post list, sorted newest-first.
3. **Writes `sitemap.xml`** with all five static pages plus every blog post (`lastmod` = today for static pages, post date for posts).
4. **Writes `feed.xml`** — RSS 2.0 feed of all posts.
5. **Writes `llms.txt`** — a markdown index of the site for LLM crawlers (per the [llms.txt convention](https://llmstxt.org/)).

Posts are sorted by their `date` front-matter (newest first). The script is idempotent — run it as many times as you like.

## Adding a book

1. Drop a cover image into `img/books/<slug>.jpg` (portrait, ~600×900 looks best in the grid).
2. Add a `<li>` to the grid in `bookshelf.html`:

   ```html
   <li>
     <a href="https://amazon.../dp/ASIN" target="_blank" rel="noopener" title="Title — Author">
       <img src="img/books/slug.jpg" alt="Title by Author">
       <span class="book-caption">Title<span class="book-author">Author</span></span>
     </a>
   </li>
   ```

No build step needed for bookshelf changes.

## Adding a side project

Edit `side-projects.html` and add a new `<li class="entry">` block with the project title, link, and one-line description. No build step needed.

## SEO and discoverability

Every page ships with:

- `<title>` and `<meta name="description">`.
- `<link rel="canonical">` to the production URL.
- Open Graph + Twitter Card meta tags (used by LinkedIn, Slack, iMessage, X, etc.).
- A Google Search Console verification placeholder (`REPLACE_WITH_GSC_TOKEN`) — replace with the real token once you create the GSC property.
- `robots.txt` explicitly allows mainstream search and LLM crawlers (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, etc.) and points to `sitemap.xml`.
- `index.html` includes JSON-LD `Person` schema; blog posts include `BlogPosting`; `blog.html` includes `Blog`.
- `feed.xml` (RSS) and `llms.txt` (LLM index) are auto-generated by the build.

### Submitting to Google Search Console

1. Deploy the site.
2. Go to https://search.google.com/search-console and add a property for `paolodona.com`.
3. Choose the HTML-tag verification method, copy the token, and replace `REPLACE_WITH_GSC_TOKEN` across the HTML files (search-and-replace).
4. Redeploy and click "Verify" in GSC.
5. In **Sitemaps**, submit `https://paolodona.com/sitemap.xml`.
6. Optional: also submit `paolodona.com` to https://www.bing.com/webmasters (covers Bing, DuckDuckGo, and ChatGPT search).

## Hosting

The site is plain static files — any static host works (GitHub Pages, Netlify, Cloudflare Pages, S3 + CloudFront…). The included `CNAME` file is for GitHub Pages with the custom `paolodona.com` domain.
