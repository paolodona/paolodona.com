# AGENTS.md

Guidance for Codex when working in this repository.

## Overview

Personal website (paolodona.com) — plain static HTML/CSS with a small Node build step that compiles markdown blog posts to HTML and regenerates `sitemap.xml`, `feed.xml`, `llms.txt`, and `llms-full.txt`. Deployed via GitHub Pages from `master` (CNAME points to `paolodona.com`).

The site is **multi-page**, not single-page. The Tumblr origin is historical — anything that mentions Tumblr is legacy and not part of the live site.

## Project structure

```
index.html              About (home)
work.html               Career history
side-projects.html      Side projects
bookshelf.html          Books — hand-edited <ul class="book-grid">
blog.html               Blog index — post list regenerated by the build script
404.html
robots.txt              Allow-lists AI crawlers; points to llms.txt
posts/YYYY-MM-DD-slug.md   Source markdown (with frontmatter: title, date, slug, summary)
blog/<slug>.html        ⚙ generated post HTML
sitemap.xml             ⚙ generated
feed.xml                ⚙ generated (RSS 2.0)
llms.txt                ⚙ generated (LLM site index)
llms-full.txt           ⚙ generated (full corpus for LLM ingestion)
css/custom.css          All styles
js/lightbox.js          Lightbox for in-post images
img/paolodona.png       Headshot — also used as favicon + OG image
img/books/<slug>.jpg    Book covers for the bookshelf grid
scripts/build-blog.mjs  Build script (gray-matter + marked)
```

Files marked ⚙ are **regenerated** by `npm run build` — do not hand-edit them.

`download-tumblr.js`, `index-tumblr-original.html`, `menu.json` are legacy artefacts kept for reference only.

## Commands

```bash
npm install          # one-time
npm run build        # regenerate blog/, sitemap, feed, llms.txt, llms-full.txt
npm run serve        # serve on http://localhost:8000 (the site uses absolute paths
                     # like /css/custom.css, so it does not work via file:// — always
                     # serve over HTTP)
```

There is no test suite, no linter, and no CI.

## Conventions

### Adding a blog post
1. Create `posts/YYYY-MM-DD-<slug>.md` with frontmatter: `title`, `date` (ISO), `slug`, `summary`.
2. Write markdown body. Inline images belong under `img/blog/<slug>/`.
3. Run `npm run build`. This writes `blog/<slug>.html`, updates `blog.html`, and regenerates `sitemap.xml`, `feed.xml`, `llms.txt`, `llms-full.txt`.
4. Commit the markdown source **and** all generated files together.

### Adding a book to the bookshelf
Use the `add-book` skill in `.Codex/skills/add-book.md`. It enforces link preference (publisher → amazon.co.uk → amazon.com), cover-image download, slug naming, exact markup, and top-of-grid insertion.

### SEO / LLM discoverability
- Every page has its own `<title>`, `<meta name="description">`, OG/Twitter tags, canonical URL, and JSON-LD.
- The home page declares `Person` (`@id: https://paolodona.com/#paolo`) and `WebSite` (`@id: .../#website`). Other pages reference these by `@id` — keep that pattern when adding new pages.
- Blog posts are emitted with `BlogPosting` + `BreadcrumbList` JSON-LD by the build script.
- `robots.txt` allow-lists GPTBot, ClaudeBot, PerplexityBot, etc. and advertises `llms.txt` / `llms-full.txt`.

### Style
- Tabs for indentation in HTML/CSS/JS.
- Use HTML entities for accented characters in HTML (`Don&agrave;`, `&rsquo;`, `&mdash;`). Markdown source can use literal Unicode.
- Keep dependencies minimal — no frameworks, no build step beyond what's already in `scripts/`.
